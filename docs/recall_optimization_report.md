## 过拟合风险判断

这次操作本身不会让模型“记住训练集”，因为没有训练权重，也没有把测试集样本用于反向传播。风险主要来自“阈值选择是否被某一个评估集过度调参”。为降低这个风险，本次同时查看了 validation 与 test 的方向是否一致：

- Validation：FN 明显下降，Dice 轻微下降
- Test：FN 同样明显下降，Dice 同样轻微下降

两个集合趋势一致，因此 `0.36` 可以作为当前的 recall-safe 部署阈值。

## 使用建议

当前配置适合偏召回的筛查型输出：宁可多标一点可疑区域，也尽量少漏掉目标区域。若后续要面向更强调标注精度的场景，可以提供两个预设：

- `balanced`：阈值 `0.40`，综合 Dice 稍高
- `recall_safe`：阈值 `0.36`，漏检更少

下一步建议在前端增加“分析模式”切换，让用户可以在“均衡模式”和“低漏检模式”之间选择，而不是把阈值隐藏在后端配置里。
